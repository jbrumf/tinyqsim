Example 4: Quantum Phase Estimation (QPE)

Purpose

Quantum Phase Estimation (QPE) is an important part of many quantum algorithms, such as Shor's factorization algorithm. This example explains how QPE works and how it can be implemented uisng TinyQsim.

It would be useful to be familiar with Phase Kickback and the Quantum Fourier Transform (QFT) from the previous example notebooks.

Introduction

The aim of QPE is to find the phase associated with an eigenvector \(\ket{\psi}\) of a unitary \(U\).

Phase Kickback

We will quickly recap the main points from the Phase Kickback notebook.

Any unitary operator can be expressed in terms of the phases \(\phi_k\) of its eigenvectors:

\(\qquad U=\sum_k{e^{i\phi_k}\,\ket{\psi_k}\bra{\psi_k}}\)

where \(\lambda_k = e^{i\phi_k}\) are the corresponding eigenvalues.

The eigenvalue always has a magnitude of one because the operator is unitary.

If we apply the unitary to one of its eigenvectors \(\ket{\psi_k}\), the result will be the corresponding eigenvalue \(e^{i\phi_k}\) multiplied by the eigenstate:

\(\qquad\begin{align*} U \ket{\psi_k} &= ({e^{i\phi_k}\,\ket{\psi_k}\bra{\psi_k}})\ \ket{\psi_k}\\ &= e^{i\phi_k}\ket{\psi_k} \end{align*}\)

If we apply a controlled unitary to one of its eigenvectors, with the control in an equal superposition state, the phase is "kicked back" to the control qubit and the target qubit is left unchanged.

Fractional Phase

The principal value of the phase \(\phi\) can be expressed as a fraction \(\theta\) of \(2\pi\).

\[\theta=\frac{\phi}{2\pi},\quad\text{where }\theta\in[0,1) \]

This can be approximated as an \(m\)-bit binary fraction:

\[\quad\theta\approx\sum_{k=1}^m{\theta_k} 2^{-k},\quad\text{where }\theta_k\in\{0,1\} \]

which can be written using '.' for the binary point, analogous to a decimal point:

\[\quad\theta\approx 0.\theta_1\theta_2\dots\theta_m \]

Encoding the phase this way allows us to represent it using a qubit register that will give the phase as a binary fraction when measured. For example, a measurement outcome of \(\ket{011}\) would correspond to the phase \(\theta=.011\) in binary, if we place a binary point before it.

Example Unitary Operator

To demonstrate the QPE circuit, we will define a function to create a one-qubit unitary \(U\) for a given phase angle \(\phi\), expressed as a fraction \(\theta\) of \(2\pi\):

\[\phi = 2 \pi \theta,\quad\text{where }\theta\in [0,1) \]

The function returns \(U^k\) where the exponent \(k\) is given as a parameter.

# Create unitary U^k with fractional phase 'theta' in [0,1]
def create_unitary(theta, k, nqt):
    assert nqt == 1
    phi = 2 * pi * theta * k
    return np.array([[1,0],[0,exp(1j * phi)]])

The QPE Circuit

We can define a function to build the QPE circuit as follows. It is assumed here that the eigenvector of interest is \(\ket{1}\), e.g. \(\ket{0001}\)

def build_qpe_circuit(unitary, nqc, nqt, omit_iqft=False):
    """Build Quantum Phase Estimation (QPE) circuit.
    :param unitary: function returning the unitary matrix
    :param nqc: Number of control qubits
    :param nqt: Number of target qubits
    :param omit_iqft: omit the inverse QFT
    :return: QPE circuit with 'nqc' control qubits and 'nqt' target qubits."""
    # Assumes eigenvector is |1>

    # Registers
    regC = range(nqc)  # Control register
    regT = range(nqc, nqc + nqt)  # Target register

    # Qubit labels
    labels = [f'C{i}' for i in range(nqc)] + [f'T{i}' for i in range(nqt)]
    label_dict = dict(zip(range(nqc + nqt), labels))

    # Create the circuit
    qc = QCircuit(nqc + nqt)
    qc.qubit_labels(label_dict, numbers=False)

    # Initialialize registers
    qc.h(regC)  # Create superposition
    qc.x(regT[nqt - 1])  # Eigenvector
    qc.barrier('1')

    # Apply unitary operators
    for i, j in enumerate(reversed(regC)):
        u = unitary(2**i, nqt)
        qc.cu(u, f'$U^{{{2**i}}}$', j, *regT)
    qc.barrier('2')

    # Inverse QFT
    if not omit_iqft:
        iqft(qc, nqc)
    return qc

The following diagram shows the general form of the QPE circuit. In this example there are 3 control qubits and 2 target qubits.

def dummy_unitary(k, nqt):
    return np.eye(2**nqt)

qc = build_qpe_circuit(dummy_unitary, 3, 2)
qc.barrier('3')
qc.measure(*range(3))
qc.draw()

The circuit has four stages, which are separated by barrier symbols in the diagram:

  • Initialisation
  • Phase kickback
  • Inverse QFT
  • Measurement

We can omit the measurement stage for a simulation, because it is possible to calculate measurement probabilities directly.

The example has a two quantum registers: a Control register C with 3 qubits and Target register T with 2 qubits.

For now, we will ignore the inverse QFT and just concentrate on the first two stages. We will also use a one-qubit target register as this is sufficient for understanding how the algorithm works.

qc = build_qpe_circuit(dummy_unitary, 3, 1, omit_iqft=True)
qc.draw()

You may recognize this as being three copies of the Phase Kickback circuit, which was discussed in a previous noteboook, each with a Hadamard gate and a unitary gate. The X gate initializes the target register to \(\ket{1}\), which is the eigenvector of \(U\) for which we wish to find the phase. The three Hadamard gates perform a Hadamard Transform that places the control register in an equal superposition of its eight basis states.

The unitaries can be cascaded because they have no effect on the state of the target register in a phase kickback circuit. The first unitary kicks back phase \(\phi\) to control qubit C2. The second unitary gate is \(U^2\), which is equivalent to two sequential \(U\) gates, so the phase kicked back to C1 is \(2\phi\). Similarly, the third unitary \(U^4\) kicks back \(4\phi\) to qubit C0.

Consider one of the basis states of the superposition, for example \(\ket{010}\). Unitary \(U^2\) kicks back \(2\phi\) to C1 but the other two unitaries have no effect since their controls are in state \(\ket{0}\). Similarly, if we consider the basis state \(\ket{110}\) then \(U^4\) kicks back \(4\phi\) to C0 and \(U^2\) kicks back \(2\phi\) to C1, but \(U^1\) has no effect.

When we consider the superpostion of all eight basis states, each one kicks back a total phase equal to \(\phi\) multiplied by the index of its basis state. The superposition results in the phases on each qubit being summed over all the basis states.

The label of a basis state is its index in the state vector expressed in binary. So \(\ket{011}\) is element 3 (011 binary) of the state vector. The 3-qubit control register accumulates the phase in binary, with C0 being the most significant bit/qubit. The phase kicked back is proportional to the weight of the binary bit.

So the total phase kicked back to each basis state is equal to \(\phi\) multiplied by its index in the state vector. If you recall the earlier notebook on the QFT, the QFT of a single basis state is a linear phase ramp with the phase proportional to the index in the Fourier basis. Consequently, if we take the inverse QFT of the output of our phase kickback circuit, we will get a single basis state whose index is the phase in binary, as a fraction of \(2\pi\).

The complete Quantum Phase Estimation Circuit is then:

nqc = 3  # Number of control qubits
qc = build_qpe_circuit(dummy_unitary, nqc, 1)
qc.draw()

Let's try it, using a fractional phase \(\theta=\frac{3}{8} = .011_2\)

theta = 3 / (2**nqc)
qc = build_qpe_circuit(partial(create_unitary, theta), nqc, 1)
qc.plot_probabilities(*range(nqc), height=0.7)

The probabilities show that measuring the control register would give the result \(011\). If we precede this by a binary point, it is the phase as the binary fraction \(\theta=.011_2\), which is correct. To get the phase in radians, we would multiply this fraction by \(2\pi\).

Let's try again, this time with a phase of \(\theta=\frac{6}{8}\), which is \(.110_2\) in binary.

theta = 6 / (2**nqc)
qc = build_qpe_circuit(partial(create_unitary, theta), nqc, 1)
qc.plot_probabilities(*range(nqc), height=0.7)

Again, we can see that the circuit is behaving as intended.

Exploring the Phase

We can plot the phases in the Fourier domain, which may help to clarify the explanation of the algorithm given above.

It is not in general meaningful to plot the phase of a subset of qubits, such as the control register, unless the state is separable such that those qubits are not entangled with the rest of the state. However, with phase kickback, the states of the control and target registers are separable, so we can define a function to plot just the control register.

def plot_phase(qc, nq):
    """Plot fractional phase of first 'nq' qubits assuming 
       they are not entangled with the rest of the state."""
    state = qc.state_vector.reshape(2**nq,-1)[:,1]
    nq = quantum.n_qubits(state)
    phase = np.atan2(state.imag, state.real) / np.pi
    phase = np.where(phase<0, phase+2, phase) / 2
    plotting.plot_bars(quantum.basis_names(nq), [phase], ylabels=[f'Phase'],ylims=[(0,1)])

So that we can examine the phase in Fourier basis, we will create a QPE circuit without the inverse QFT. There are 5 control qubits and the phase is \(\theta=\large\frac{1}{2^5}\) .

nqc = 5  # Number of control qubits
theta = 1 / (2**nqc)
qc = build_qpe_circuit(partial(create_unitary, theta), nqc, 1, omit_iqft=True)
plot_phase(qc, nqc)

The phase increases as a linear ramp. There are 32 steps and \(\theta=\frac{1}{32}\), so it reaches just less than 1 by the end.

Repeating this for \(\theta=\frac{4}{32}\)

nqc = 5  # Number of control qubits
theta = 4 / (2**nqc)
qc = build_qpe_circuit(partial(create_unitary, theta), nqc, 1, omit_iqft=True)
plot_phase(qc, nqc)

The phase is now increasing at 4 times the rate, so it completes 4 cycles in the length of the state vector. The apparent backward jumps are just because the phase in radians is modulo \(2\pi\).

If we now apply the inverse QFT, we get the final result, which is the basis state \(\ket{00100}\), that gives us the required phase.

iqft(qc, nqc)
qc.plot_probabilities(*range(nqc))

Reduced Resolution

In the examples above, we chose phases that could be represented exactly as a binary fraction with the number control qubits used. However, sometimes this may not be the case. For example, if \(\theta= \frac{4.5}{32}\)

nqc = 5  # Number of control qubits
theta = 4.5 / (2**nqc)
qc = build_qpe_circuit(partial(create_unitary, theta), nqc, 1, omit_iqft=True)
plot_phase(qc, nqc)


iqft(qc, nqc)
qc.plot_probabilities(*range(nqc))

The probabilities is no longer concentrated in a single basis state, but are spread out. If we were to make a measurement, there would be a high probability of getting 00100 or 00101. The Quantum Phase Estimation algorithm has given us an estimate, as the name implies.
